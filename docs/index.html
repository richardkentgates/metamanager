What Metamanager Does

PHP can't do lossless compression, video remux, or write IPTC/XMP and format-native audio tags. Metamanager can.

🗜

True Lossless Compression

JPEG files are optimised with jpegtran — no re-encoding, no quality loss, no pixel changes. PNG files are processed by optipng. WebP files are recompressed with cwebp -lossless. Files are only replaced if the result is smaller.

🎥

Video & Audio Support

Upload MP4, MOV, AVI, MKV, WebM, WMV, MP3, M4A, OGG, WAV, FLAC, and more. Metadata (title, description, creator, copyright, keywords, date) writes to each format's native tag system: QuickTime atoms for MP4/M4A, ID3 for MP3, Vorbis comments for OGG/FLAC, XMP-only for AVI/WAV/WMV/WMA. Video containers are remuxed losslessly via ffmpeg.

📌

PDF Metadata

PDF documents are first-class citizens. Title, author, keywords, and creation date are imported from embedded PDF tags on upload. WordPress fields write back as XMP. Attachment pages emit a Schema.org DigitalDocument JSON-LD block.

🏷

Full Metadata Standards

Write EXIF, IPTC, and XMP simultaneously via ExifTool — the industry standard. Every tag written in all three formats at once for maximum compatibility across editing software, search engines, and CMS platforms.

🔄

Bidirectional Metadata Sync

On upload, embedded EXIF/IPTC/XMP tags are read from the file and used to pre-populate WordPress fields automatically. User-set values are never overwritten. Edit fields in WordPress and the daemon writes them back into the file.

📍

GPS Coordinates

When clients shoot with location services enabled, Metamanager reads GPS latitude, longitude, and altitude directly from the camera's EXIF data. No manual entry — coordinates are imported and stored automatically at upload time.

�

Expanded Metadata Fields

11 custom fields per image: Creator, Copyright, Owner, Headline, Credit, Keywords, Date Created, Rating (0–5 stars), City, State/Province, Country. All stored as registered post meta, embedded into the file by the daemon, and fully exposed via the WordPress REST API.

🔍

Schema.org & Open Graph

Every attachment page automatically receives type-appropriate structured data: ImageObject for images (with GeoCoordinates when GPS data is present), VideoObject for video, AudioObject for audio, and DigitalDocument for PDF. Open Graph tags use the correct type per media family. URL-format copyright values emit <link rel="license">; plain-text values emit <meta name="copyright">.

🖥

Native Dashboard Integration

Custom fields on every media file edit screen, grouped by purpose: Attribution & Rights, Editorial, Classification, Location. A live compression-status column in the Media Library. Searchable, paginated job history. All inside WordPress — no separate admin interface to learn.

⚡

Asynchronous by Design

Image uploads never slow down. Jobs are queued as JSON files. Two inotifywait-powered daemons process them immediately in the background. Results feed back into WordPress via WP-Cron.

🔒

Bulk Without False Attribution

Bulk actions are limited to injecting Publisher and Website — neutral site provenance only. Creator, Copyright, and Owner are individual fields, intentionally never overwritten in bulk.

💚

Daemon Health Status

A status banner at the top of every Metamanager screen shows in real time whether the compression and metadata daemons are running. Detection uses a PID file — no systemctl privilege required for the web process.

🔁

Re-queue Failed Jobs

Every failed job appears in the history table with a one-click Re-queue button. No manual file hunting, no WP-CLI commands needed to recover from a temporary failure.

📦

Native Auto-Updates

Integrates with the WordPress update pipeline. New releases published to GitHub appear automatically in Dashboard → Updates — update with one click, just like any other plugin. A manual “Check for Updates” link is also available on the Plugins page.

🌐

Multisite Ready

Network activation creates the job-log table and schedules cron on every existing site. New blog creation is handled automatically via wp_initialize_site. Per-site settings, per-site tables, shared job queue — each concern is scoped correctly.

🧹

Clean Uninstall

An opt-in setting under Media → MM Settings → Data & Uninstall removes all options, attachment metadata, the job-log table, and the queue directory when the plugin is deleted. Nothing is removed by default — data is always preserved unless you explicitly choose otherwise.

How It Works

PHP coordinates. The OS executes. Clean separation of responsibility.

1

Upload or Edit

On upload, embedded metadata is read from the file into WordPress fields. On edit, changed fields are queued for writing back to the file.

→
2

Job Written

Metamanager writes a small JSON job file to the queue directory. PHP does nothing else.

→
3

Daemon Processes

inotifywait detects the new file. The daemon processes the file with jpegtran, optipng, cwebp, ffmpeg, or ExifTool depending on file type and job.

→
4

Result Recorded

The daemon writes a result JSON to the completed folder. WP-Cron picks it up every minute and records it in the database.

Keeping Metamanager Up to Date

Two methods — choose what fits your workflow.

📦

WordPress Dashboard (recommended)

Metamanager integrates with the native WordPress update system. When a new release is tagged on GitHub, it appears in Dashboard → Updates automatically within 12 hours. Click Update now — exactly the same as any other plugin.

A Check for Updates link is available next to the plugin on Plugins → Installed Plugins for an immediate check.

🖥

Server Script (plugin files only)

Use --update to sync only PHP, JS, and asset files. Daemons keep running, dependencies are not reinstalled, and systemd services are not restarted — zero downtime for the compression and metadata pipelines.

sudo bash install.sh --update

Schema.org & Open Graph

Every image page gets structured data automatically — no configuration required.

🔍

Schema.org ImageObject

A full JSON-LD block is emitted in <head> on attachment pages and any post with a featured image. Includes name, description, creator, copyright, keywords, date, location, and GPS coordinates when available. Eligible for Google Images rich results and Google Discover.

📲

Open Graph Tags

Correct og:image, og:image:alt, og:image:width, og:image:height, and og:image:type tags are written for every image. Social platforms — Facebook, LinkedIn, Slack, iMessage — use these to generate accurate link previews.

⚖️

License Link

When a copyright value is set, Metamanager emits <link rel="license"> for URL-format values (e.g. Creative Commons URIs) or <meta name="copyright"> for plain-text notices — giving crawlers and aggregators a machine-readable rights signal.

WP-CLI

Full command-line access to every Metamanager operation. WP-CLI 2.0+ required.

wp metamanager compress

Queue lossless compression for one or all compressible attachments. Images are recompressed with jpegtran, optipng, or cwebp. Video is remuxed losslessly with ffmpeg -c copy. Audio and PDF have no compression step.

wp metamanager compress             # all compressible files
wp metamanager compress all         # explicit
wp metamanager compress 42          # single attachment ID
wp metamanager compress all --force # re-queue already-compressed files
wp metamanager import

Read embedded tags from the file and populate empty WordPress fields. Existing user-set values are never overwritten. Supports images (EXIF/IPTC/XMP), video (QuickTime atoms/XMP), audio (ID3/Vorbis/XMP), and PDF (XMP).

wp metamanager import               # all supported files
wp metamanager import all           # explicit
wp metamanager import 42            # single attachment ID
wp metamanager scan

Import metadata for every library file not yet processed by Metamanager. Faster than import all on large existing libraries — already-synced files are skipped automatically. Good for first run after activation on a site with an existing media library.

wp metamanager scan
wp metamanager queue status

Print pending job counts by type and current daemon health status.

REST API

Every operation is available via the WordPress REST API. Requires upload_files capability and a valid X-WP-Nonce.

Base URL: https://yoursite.com/wp-json/metamanager/v1

Method Endpoint Description Key parameters
GET /stats Aggregate compression statistics (total jobs, completed, failed, bytes saved). —
GET /jobs Paginated job history. Pagination totals in X-WP-Total and X-WP-TotalPages headers. search, orderby, order, per_page, page
GET /jobs/{id} Single job record by database ID. Returns 404 if not found. —
GET /attachment/{id}/status Compression and metadata sync status for one attachment. —
POST /attachment/{id}/compress Queue lossless compression. Images → all sizes recompressed. Video → lossless remux. Audio/PDF → 422. force (boolean)
POST /compression-status Batch status query used by the Media Library column. Returns a map of ID → status string. ids (array)

Example: attachment status response

GET /wp-json/metamanager/v1/attachment/42/status

{
  "id": 42,
  "compression": "compressed",
  "meta_synced": true
}

Example: queue compression

POST /wp-json/metamanager/v1/attachment/42/compress
X-WP-Nonce: <nonce>

{ "id": 42, "queued": true, "message": "Compression jobs queued." }

Install in One Command

Works on Ubuntu 22.04+, Debian 12+, RHEL/Rocky 9+. Requires systemd and root access.

One-command installer (auto-detects WordPress path)
wget -qO- https://raw.githubusercontent.com/richardkentgates/metamanager/main/install.sh | sudo bash
With explicit WordPress path
sudo bash install.sh --wp-path /var/www/html
Update plugin files only (keeps daemons running, skips dependencies)
wget -qO- https://raw.githubusercontent.com/richardkentgates/metamanager/main/install.sh | sudo bash -s -- --update

What the installer does

  1. Detects your WordPress installation path
  2. Installs jpegtran, optipng, cwebp, ffmpeg, exiftool, inotify-tools, and jq via apt or dnf
  3. Copies the plugin into wp-content/plugins/metamanager/
  4. Patches daemon scripts with your actual WP_CONTENT_DIR — no hardcoded paths
  5. Installs and starts both systemd daemons as www-data
  6. Activates the plugin via WP-CLI if available

Open Source Credits

Metamanager is built on the shoulders of these outstanding open source projects. Full credit and gratitude to their authors and maintainers.

ExifTool

Phil Harvey
Artistic License / GPL

The backbone of all metadata work in Metamanager. ExifTool reads and writes EXIF, IPTC, and XMP tags across virtually every image format in existence. We use it to import embedded metadata on upload and to write all metadata fields back to the file.

exiftool.org GitHub ↗
🗜

libjpeg-turbo / jpegtran

libjpeg-turbo Project · Independent JPEG Group
BSD / IJG License

jpegtran performs lossless JPEG optimisation — reordering Huffman tables and enabling progressive scan without decoding or re-encoding a single pixel. The IJG created the original implementation; libjpeg-turbo maintains and accelerates it.

This software is based in part on the work of the Independent JPEG Group.

libjpeg-turbo.org GitHub ↗
🖼

optipng

Cosmin Truță
zlib / libpng License

optipng tries multiple DEFLATE parameters and filter combinations to find the smallest lossless PNG representation. No pixels are changed. We use -o2 -preserve to balance speed and compression while retaining all file metadata.

optipng.sourceforge.net
🌐

libwebp / cwebp

Google
BSD 3-Clause

cwebp, part of Google's libwebp package, compresses WebP images losslessly with -lossless. Files are only replaced if the result is smaller. libwebp is Google's reference implementation of the WebP image format.

developers.google.com/speed/webp Source ↗
🎬

FFmpeg

FFmpeg contributors
LGPL v2.1+ / GPL v2+

ffmpeg remuxes video containers losslessly using -c copy — stripping padding and redundant data without re-encoding a single frame. The output is bitstream-identical to the input. No transcoding, no quality change.

ffmpeg.org Source ↗
👁

inotify-tools

inotify-tools contributors
GPL v2

inotifywait is what makes the daemons instant-response rather than polling. When a job file is written, inotifywait fires immediately — no sleep loops, no latency. The Linux kernel's inotify subsystem watches; inotify-tools provides the userspace interface.

GitHub ↗

jq

Stephen Dolan · jqlang organisation
MIT License

jq is used inside the Bash daemons to parse job JSON files written by PHP — extracting field values by key without requiring any additional runtime. Lightweight, dependency-free in execution, and universally available on our supported Linux distributions.

jqlang.github.io/jq GitHub ↗
